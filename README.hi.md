## "रजिस्ट्रम" क्या है?

"रजिस्ट्रम" एक **संरचनात्मक रजिस्ट्री** है - यह एक पुस्तकालय है जो राज्य परिवर्तनों को रिकॉर्ड करता है, उन्हें मान्य करता है, और विशिष्ट सीमाओं के तहत उन्हें व्यवस्थित करता है, ताकि संरचना की व्याख्यात्मकता बनी रहे, भले ही एंट्रॉपी (अव्यवस्था) बढ़ती रहे।

| संपत्ति। | अर्थ। |
|----------|---------|
| **Structural** | यह केवल रूप पर आधारित है, अर्थ पर नहीं। |
| **Deterministic** | समान इनपुट → हमेशा समान आउटपुट, यह हमेशा सत्य होता है। |
| **Fail-closed** | अमान्य इनपुट के कारण सिस्टम पूरी तरह से खराब हो जाता है, आंशिक रूप से ठीक नहीं होता। |
| **Replayable** | ऐतिहासिक निर्णय, समान परिणामों के साथ, फिर से दोहराए जा सकते हैं। |
| **Non-agentic** | यह कभी भी कोई कार्य नहीं करता, कोई निर्णय नहीं लेता, और न ही किसी चीज़ को अनुकूलित करता है। |

"रजिस्ट्रम यह सुनिश्चित करता है कि परिवर्तन स्पष्ट रूप से दिखाई दें।"

## "रेजिस्ट्रम" क्यों?

प्रणालियाँ विकसित होती हैं। अव्यवस्था बढ़ती जाती है। संरचनाएं क्षीण होती जाती हैं।

अधिकांश उपकरण इस समस्या का समाधान बुद्धिमत्ता जोड़कर करते हैं - जैसे कि अनुकूलन उपकरण, सहायक प्रणालियाँ, और स्वतः-सुधार करने वाले परतें। लेकिन, "रजिस्ट्रम" एक विपरीत दृष्टिकोण अपनाता है: यह **बाधाएं (constraints) जोड़ता है।**

- **पुस्तकालय के लेखकों के लिए:** राज्य प्रबंधन में संरचनात्मक गारंटी को शामिल करें, ताकि उपयोगकर्ता बिना किसी अतिरिक्त प्रयास के पठनीयता का लाभ उठा सकें।
- **ऑडिट के लिए महत्वपूर्ण प्रणालियों के लिए:** प्रत्येक राज्य परिवर्तन निश्चित, दोहराने योग्य और स्वतंत्र रूप से सत्यापित करने योग्य होता है।
- **उन टीमों के लिए जो जटिलता से बचना चाहती हैं:** रजिस्ट्रम, अमान्य परिवर्तनों को चुपचाप अनदेखा करने के बजाय, संरचित प्रतिक्रियाओं के साथ अस्वीकार करता है।

परिणाम: एक ऐसी प्रणाली जो बनाई गई है, जिसमें पहचान, वंश और क्रम हमेशा जांचने योग्य रहते हैं, चाहे उसमें कितनी भी बदलाव क्यों न आए हों।

## मूल सिद्धांत।

"अव्यवस्था स्वीकार्य है, लेकिन अस्पष्टता नहीं।"

"रजिस्ट्रम" वैश्विक स्तर पर एंट्रॉपी (अव्यवस्था) को कम नहीं करता है।

यह केवल उन स्थानों को सीमित करता है जहाँ एंट्रॉपी मौजूद हो सकती है, ताकि समय के साथ पहचान, वंश और संरचना की जांच की जा सके।

## "रजिस्ट्रम" क्या नहीं है।

"Registrum" स्पष्ट रूप से निम्नलिखित नहीं है:

- एक अनुकूलक, अभिकर्ता, या निर्णय लेने वाला।
- एक नियंत्रक, अनुशंसाकर्ता, या बुद्धिमत्ता।
- अनुकूलनशील, सीखने वाला, या स्वतः ठीक होने वाला।

यह कभी भी *महत्वपूर्ण प्रश्नों* का उत्तर नहीं देता।
यह केवल उन परिस्थितियों को बनाए रखता है जिनके तहत उन प्रश्नों का उत्तर देना संभव हो सकता है।

## "रजिस्ट्रम" कैसे काम करता है।

### संरचनात्मक रजिस्ट्रार।

पंजीयक एकमात्र संवैधानिक प्राधिकारी है:

- यह सभी अवस्था परिवर्तनों को **11 संरचनात्मक नियमों** के विरुद्ध सत्यापित करता है।
- यह पहचान, उत्पत्ति और क्रम से संबंधित नियमों का पालन सुनिश्चित करता है।
- यह निश्चितता और पता लगाने की क्षमता की गारंटी देता है।
- यह नियमों के उल्लंघन को दिखाता है, लेकिन उन्हें ठीक नहीं करता (यह "विफलता-सुरक्षित" है)।

सब कुछ इसी के माध्यम से दर्ज होता है। कुछ भी इससे आगे नहीं जा पाता।

### ग्यारह अपरिवर्तनीय गुण।

| वर्ग। | गिनती। | उद्देश्य। |
|-------|-------|---------|
| **Identity** | 3 | अद्वितीय, अपरिवर्तनीय, सामग्री-आधारित पता। |
| **Lineage** | 4 | मान्य वंशानुक्रम, चक्रीय नहीं, पता लगाने योग्य। |
| **Ordering** | 4 | एकसमान, अंतराल-रहित, और निश्चित। |

ये अपरिवर्तनीय तत्व हैं, जो संविधान में निहित हैं। इनमें बदलाव करने के लिए औपचारिक प्रक्रियाओं का पालन करना आवश्यक है।

## दोहरे संवैधानिक गवाह।

"रजिस्ट्रम" दो स्वतंत्र और अपरिवर्तनीय इंजन (इनवेरिएंट इंजन) का उपयोग करता है:

| गवाह। | भूमिका। | कार्यान्वयन। |
|---------|------|----------------|
| **Registry** | प्राथमिक अधिकार। | संकलित डीएसएल (आरपीजी संस्करण 1) |
| **Legacy** | गवाह (जो प्रत्यक्षदर्शी नहीं है)। | टाइपस्क्रिप्ट में प्रेडिकेट (Predicate) क्या होते हैं। |

एच चरण तक, **रजिस्ट्री (registry) ही संवैधानिक प्रणाली का प्राथमिक आधार है।**
पुराने सिस्टम (लेगेसी सिस्टम) एक स्वतंत्र, द्वितीयक सत्यापनकर्ता के रूप में मौजूद रहेंगे।

### दो गवाहों की आवश्यकता क्यों होती है?

- **समझौता आवश्यक है** — किसी भी बदलाव को मान्य होने के लिए, दोनों पक्षों को सहमत होना होगा।
- **असहमति प्रक्रिया को रोकती है** — असमानता (parity) में विचलन सिस्टम को बंद कर देता है (यह एक "सुरक्षित विफलता" प्रणाली है)।
- **स्वतंत्रता जानबूझकर बनाई गई है** — न तो कोई भी दूसरे को नियंत्रित कर सकता है।

यह एक सुरक्षा और पठनीयता सुविधा है, तकनीकी ऋण नहीं।
ड्युअल-मोड अनिश्चित है। न तो किसी भी 'विटनेस' को हटाने की कोई योजना है।

### समानता का प्रमाण

274 परीक्षण व्यवहारिक समानता साबित करते हैं:
- सभी अपरिवर्तनीय वर्गों में 58 समानता परीक्षण
- 12 निरंतरता समानता परीक्षण (समय की स्थिरता)
- रीप्ले समानता: लाइव निष्पादन ≡ रीप्ले किया गया निष्पादन

## बाहरी प्रमाण (वैकल्पिक)

Registrum वैकल्पिक रूप से सार्वजनिक सत्यापन के लिए एक बाहरी अपरिवर्तनीय लेज़र (जैसे XRPL) पर क्रिप्टोग्राफिक प्रमाण जारी कर सकता है।

| संपत्ति। | मूल्य |
|----------|-------|
| डिफ़ॉल्ट | अक्षम |
| अधिकार | गैर-अधिकार (केवल विटनेस) |
| व्यवहार पर प्रभाव | कोई नहीं |

प्रमाणपत्र रिकॉर्ड करते हैं कि Registrum ने क्या निर्णय लिया।
Registrum यह तय करता है कि क्या मान्य है।

**अधिकार अंदर की ओर प्रवाहित होता है। विटनेस बाहर की ओर प्रवाहित होता है।**

देखें:
- [`docs/WHY_XRPL.md`](docs/WHY_XRPL.md) — तर्क
- [`docs/ATTESTATION_SPEC.md`](docs/ATTESTATION_SPEC.md) — विनिर्देश

## शुरुआत कैसे करें

### स्थापना

```bash
npm install @mcp-tool-shop/registrum
```

### त्वरित शुरुआत

```typescript
import { StructuralRegistrar } from "@mcp-tool-shop/registrum";

const registrar = new StructuralRegistrar({ mode: "legacy" });

// Register a root state
const result = registrar.register({
  from: null,
  to: { id: "state-1", structure: { version: 1 }, data: {} },
});

if (result.kind === "accepted") {
  console.log(`Registered at index ${result.orderIndex}`);
} else {
  // Structured refusal — the violations name which invariants failed
  console.log(`Refused: ${result.violations.map((v) => v.invariantId)}`);
}
```

### मोड

| मोड | इंजन | कब उपयोग करें |
|------|--------|-------------|
| `"legacy"` | टाइपस्क्रिप्ट में प्रेडिकेट (Predicate) क्या होते हैं। | त्वरित प्रोटोटाइपिंग, कोई बाहरी निर्भरता नहीं |
| `"registry"` (डिफ़ॉल्ट) | संकलित RPEG v1 DSL | पूर्ण ड्युअल-विटनेस के साथ उत्पादन उपयोग |

```typescript
import { StructuralRegistrar } from "@mcp-tool-shop/registrum";
import { loadCompiledRegistry } from "@mcp-tool-shop/registrum/registry";

// Registry mode (default) — compiled DSL + legacy as dual witnesses
const compiledRegistry = loadCompiledRegistry();
const registrar = new StructuralRegistrar({ compiledRegistry });
```

## API त्वरित संदर्भ

### मुख्य निर्यात

```typescript
// Implementation
import { StructuralRegistrar } from "@mcp-tool-shop/registrum";

// Types
import type {
  State,          // { id, structure, data }
  Transition,     // { from, to, metadata? }
  RegistrationResult,   // { kind: "accepted" | "refused", ... }
  Invariant,      // { id, scope, check }
  InvariantViolation,   // { invariantId, classification, message }
} from "@mcp-tool-shop/registrum";

// Invariants
import {
  INITIAL_INVARIANTS,     // All 11 invariants
  getInvariantsByScope,   // Filter by "identity" | "lineage" | "ordering"
  getInvariantById,       // Lookup by invariant ID
} from "@mcp-tool-shop/registrum";

// Version
import { REGISTRUM_VERSION } from "@mcp-tool-shop/registrum";
```

### `StructuralRegistrar`

| विधि | वापसी | विवरण |
|--------|---------|-------------|
| `register(transition)` | `RegistrationResult` | एक राज्य संक्रमण को मान्य और रिकॉर्ड करें |
| `getState(id)` | `State \ | अपरिभाषित` | आईडी द्वारा पंजीकृत राज्य प्राप्त करें |
| `getHistory()` | `Transition[]` | स्वीकृत संक्रमणों का पूर्ण क्रमबद्ध इतिहास |
| `snapshot()` | `RegistrarSnapshotV1` | नियतात्मक, कंटेंट-एड्रेस्ड स्नैपशॉट |

## शासन

Registrum एक **संवैधानिक मॉडल** के तहत शासित है।

| सिद्धांत | अर्थ। |
|-----------|---------|
| व्यवहार संबंधी गारंटी > सुविधा वेग | सहीता को प्राथमिकता |
| केवल साक्ष्य-आधारित परिवर्तन | बिना प्रमाण के कोई परिवर्तन नहीं |
| औपचारिक प्रक्रिया आवश्यक है | प्रस्ताव, कलाकृतियाँ, निर्णय |

### वर्तमान स्थिति

- **फेज एच**: पूर्ण (रजिस्ट्री डिफ़ॉल्ट, प्रमाण सक्षम)
- **शासन**: सक्रिय और लागू
- **सभी परिवर्तन**: औपचारिक शासन प्रक्रिया की आवश्यकता है

भविष्य में सभी परिवर्तन शासन निर्णय हैं, इंजीनियरिंग कार्य नहीं।

## सुरक्षा और डेटा दायरा।

| पहलू | विवरण |
|--------|--------|
| **Data touched** | मेमोरी में स्थिति परिवर्तन, वैकल्पिक JSON स्नैपशॉट स्थानीय फ़ाइल सिस्टम में। |
| **Data NOT touched** | कोई नेटवर्क अनुरोध नहीं, कोई बाहरी एपीआई नहीं, कोई डेटाबेस नहीं, कोई उपयोगकर्ता क्रेडेंशियल नहीं। |
| **Permissions** | केवल उपयोगकर्ता द्वारा निर्दिष्ट स्नैपशॉट पथों पर पढ़ें/लिखें (जब दृढ़ता का उपयोग किया जाता है)। |
| **Network** | कोई नहीं — पूरी तरह से ऑफ़लाइन लाइब्रेरी (XRPL प्रमाणीकरण डिफ़ॉल्ट रूप से अक्षम)। |
| **Telemetry** | कोई भी एकत्र या भेजा नहीं गया। |
